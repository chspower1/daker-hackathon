# 핵심 기능 명세

## 1. 홈

- 홈은 랜딩 허브 역할을 수행한다.
- `해커톤 보러가기`, `팀 찾기`, `랭킹 보기` CTA 3개를 제공한다.
- 읽기 전용 진입점이며 프로필 없이 접근 가능해야 한다.

## 2. 해커톤 목록

- 공개 해커톤 목록을 카드 형태로 노출한다.
- 기본 표시 항목은 제목, 상태, 태그, 일정이며 참가자 수는 데이터가 있을 때만 표시한다.
- 카드 클릭 시 상세 페이지로 이동한다.
- 목록이 비어 있거나 로딩 실패 시 상태 화면으로 분기한다.

## 3. 해커톤 상세

- 선택한 해커톤의 공개 정보를 한 화면에 모아 보여준다.
- Overview, Info / Notice, Eval, Schedule, Prize, Teams, Submit, Leaderboard 의 8개 섹션을 포함한다.
- 일부 상세 데이터가 비어 있어도 페이지 전체를 실패시키지 않고, 가능한 공개 정보부터 렌더링한다.

### Teams

- 현재 해커톤에 연결된 모집글을 보여준다.
- 기본 데이터는 공개 시드와 `localStorage.teams` 병합 결과를 사용한다.
- `/camp?hackathon={slug}` 링크를 제공한다.
- 모집글 생성 전에는 로컬 프로필 존재를 확인한다.

### Submit

- 해커톤 설정에 맞는 입력 필드를 동적으로 구성한다.
- 저장은 `draft`, 최종 제출은 `submitted` 상태로 관리한다.
- 제출 데이터에는 `profileId`, `profileNicknameSnapshot`, `notes`, 산출물 정보를 함께 저장한다.
- 최종 제출 후에는 리더보드 반영 상태가 함께 유지되어야 한다.

### Leaderboard

- 팀명, 점수, 제출 시각, 점수 breakdown, 공개 산출물 링크를 표시한다.
- `미제출` 항목은 순위를 부여하지 않는다.
- 내부 식별자나 비공개 값은 표시하지 않는다.

## 4. 캠프

- 전체 모집글 조회와 작성 기능을 제공한다.
- `hackathon` 쿼리가 있으면 해당 해커톤 모집글만 필터링한다.
- 작성 폼은 `name`, `intro`, `isOpen`, `lookingFor[]`, `contact.url`, `hackathonSlug` 를 다룬다.
- 작성자 입력 필드는 별도로 받지 않고 활성 로컬 프로필을 사용한다.
- 작성 결과는 새로고침 후에도 유지되어야 한다.

## 5. 랭킹

- 글로벌 공개 닉네임 기준 랭킹을 테이블로 보여준다.
- 최소 컬럼은 `rank`, `nickname`, `points` 다.
- `localStorage.rankings` 변경이 반영되어야 한다.
- `localProfile` 변경이 랭킹 닉네임에 자동 반영되지는 않는다.

## 6. 공통 기능 규칙

- 읽기 화면은 프로필 없이 동작한다.
- 쓰기 동작은 프로필 생성 또는 재사용 이후에만 허용한다.
- 공개 가능한 데이터만 표시한다.
- 모든 주요 화면은 한국어/영어 전환과 새로고침 유지가 동작해야 한다.
