# 주요 사용 흐름

아래 흐름은 서비스의 핵심 사용 흐름이다.

## 1. 홈에서 해커톤 탐색 후 제출

1. 사용자가 홈에서 `해커톤 보러가기` CTA를 누른다.
2. 해커톤 목록에서 원하는 이벤트를 선택한다.
3. 상세 페이지에서 규칙, 일정, 평가 기준, 팀 모집 상황을 확인한다.
4. 제출 저장 또는 최종 제출을 시도한다.
5. 프로필이 없으면 먼저 로컬 프로필을 만들고, 이후 제출 상태와 리더보드 반영 결과를 유지한다.

## 2. 해커톤 상세에서 팀 모집으로 이동

1. 사용자가 상세 페이지의 Teams 섹션을 본다.
2. `camp` 연결 CTA를 통해 `/camp?hackathon={slug}` 로 이동한다.
3. 현재 해커톤 모집글만 확인하거나 새 모집글을 작성한다.
4. 새 모집글은 활성 프로필 스냅샷과 함께 저장되고 새로고침 후에도 유지된다.

## 3. 홈 또는 내비게이션에서 랭킹 확인

1. 사용자가 홈 CTA 또는 상단 내비게이션으로 `/rankings` 에 진입한다.
2. 공개 닉네임 기준 랭킹 테이블을 본다.
3. 랭킹 데이터 변경 시 화면이 최신 상태를 다시 읽는다.

## 4. 언어 전환과 지속성

1. 사용자가 언어 전환 UI에서 한국어 또는 영어를 선택한다.
2. 주요 카피, 표 헤더, 접근성 문구, 메타데이터가 선택 언어 기준으로 바뀐다.
3. 새로고침 후에도 마지막 선택 언어가 유지된다.
