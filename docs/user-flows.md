# 주요 사용 흐름

## 1. 홈에서 해커톤 탐색으로 이동

1. 사용자가 `/` 에 접속해 서비스 소개와 주요 CTA를 확인한다.
2. 사용자가 `해커톤 탐색하기` 를 눌러 `/hackathons` 로 이동한다.
3. 해커톤 목록에서 상태와 태그 필터를 조합해 원하는 이벤트를 찾는다.
4. 사용자가 관심 있는 카드를 눌러 `/hackathons/[slug]` 상세 페이지로 이동한다.

예외 처리

- 목록 데이터가 없으면 빈 상태 화면으로 현재 참여 가능한 이벤트가 없음을 안내한다.
- 목록 읽기에 실패하면 오류 상태 화면으로 전환해 정상 로딩이 되지 않았음을 알려준다.

## 2. 해커톤 상세 확인 후 캠프로 이동

1. 사용자가 상세 페이지에서 개요, 공지, 평가, 일정, 상금 정보를 확인한다.
2. 같은 화면의 `팀` 섹션에서 현재 연결된 모집글 수와 팀 목록을 본다.
3. 사용자가 `캠프로 이동` 버튼을 눌러 `/camp?hackathon={slug}` 로 이동한다.
4. 캠프 페이지에서 해당 해커톤 모집글만 필터링된 상태로 확인한다.
5. 사용자가 필요하면 모집글 작성 마법사 모달을 열고 새 팀 모집글을 작성한다.

예외 처리

- 상세 정보 일부가 없으면 해당 섹션만 빈 상태로 보이고, 다른 섹션 탐색은 계속할 수 있다.
- 해당 해커톤 모집글이 없으면 캠프 페이지에서 빈 상태 메시지로 첫 모집글 작성을 유도한다.
- 프로필 없이 작성을 시작하면 모달의 첫 단계에서 프로필 생성 안내가 표시된다.

## 3. 프로필 생성 후 제출 초안 저장과 최종 제출

1. 사용자가 해커톤 상세의 `Submit` 섹션으로 이동한다.
2. 프로필이 없으면 닉네임을 입력해 로컬 프로필을 생성한다.
3. 사용자가 팀 이름, 산출물 링크, 메모를 입력한다.
4. `Save Draft` 를 누르면 현재 제출 내용이 초안으로 저장된다.
5. 사용자가 내용을 보완한 뒤 `Final Submit` 을 누른다.
6. 제출 상태가 `submitted` 로 바뀌고, 같은 팀명이 해커톤 리더보드에도 반영된다.

예외 처리

- 닉네임이 비어 있으면 프로필 생성이 완료되지 않고 안내 메시지만 유지된다.
- 필수 입력이 비어 있으면 브라우저 기본 검증으로 제출이 진행되지 않는다.
- 제출 저장 또는 리더보드 갱신에 실패하면 성공 상태로 보이지 않고 오류 메시지를 보여준다.

## 4. 랭킹 확인과 언어 전환 유지

1. 사용자가 홈 CTA 또는 상단 내비게이션으로 `/rankings` 에 진입한다.
2. 공개 닉네임 기준의 랭킹 테이블에서 순위와 점수를 확인한다.
3. 사용자가 언어 전환 UI에서 한국어 또는 영어를 선택한다.
4. 페이지 제목, 설명, 표 헤더, 주요 문구가 선택한 언어에 맞게 바뀐다.
5. 새로고침 후에도 마지막 언어 선택이 유지된 상태로 다시 진입한다.
6. 같은 탭이나 다른 탭에서 `rankings` 저장소 값이 바뀌면 열린 화면도 최신 값으로 다시 정렬된다.

예외 처리

- 랭킹 데이터가 없으면 빈 상태 화면으로 시즌 데이터가 아직 없음을 안내한다.
- 랭킹 읽기에 실패하면 오류 상태와 재시도 가능한 화면을 보여준다.
- 클라이언트 라우트 로딩이 지연되면 로딩 상태를 먼저 노출해 빈 화면을 남기지 않는다.
