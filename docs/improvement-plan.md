# 개발 및 개선 계획

현재 버전은 핵심 사용자 흐름을 검증하는 프로토타입이다. 기본 구현을 마친 뒤에는 사용성 보강, 데이터 모델 확장, 운영 품질 강화 순서로 개선한다.

## 우선순위와 일정

| 우선순위 | 목표 시점 | 개선 항목 | 기대 효과 |
|---|---|---|---|
| 높음 | 1차 정리 단계 | 상태 화면과 빈 상태 메시지 세분화 | 정보 없음, 저장 실패, 잘못된 진입 상황을 더 쉽게 이해할 수 있다. |
| 높음 | 1차 정리 단계 | 해커톤 목록 필터와 캠프 필터 사용성 개선 | 사용자가 원하는 이벤트와 팀 모집글을 더 빠르게 찾을 수 있다. |
| 높음 | 1차 정리 단계 | 제출 폼 입력 안내와 검증 메시지 보강 | 초안 저장/최종 제출 과정에서 입력 실수를 줄일 수 있다. |
| 중간 | 2차 기능 확장 | 캠프 모집글 수정, 모집 마감 처리 | 팀 모집 상태를 실제 운영 흐름에 더 가깝게 관리할 수 있다. |
| 중간 | 2차 기능 확장 | 제출 이력과 버전 관리 | 사용자가 제출 변경 내역을 확인하고 되돌릴 수 있다. |
| 중간 | 2차 기능 확장 | 해커톤별 리더보드 정책 세분화 | 대회별 점수 규칙과 공개 방식 차이를 반영할 수 있다. |
| 낮음 | 3차 범위 확장 | 인증, 계정 기반 동기화, 백엔드 연동 | 브라우저 단위 프로토타입을 실제 서비스 구조로 확장할 수 있다. |
| 낮음 | 3차 범위 확장 | 관리자 화면, 운영 도구, 실시간 협업 | 운영 편의성과 서비스 확장성을 높일 수 있다. |

## 단계별 계획

### 1단계: 현재 범위 완성도 강화

- 해커톤 목록의 상태/태그 탐색 경험을 더 명확하게 다듬는다.
- 캠프와 제출 화면의 입력 오류 안내를 구체적으로 보강한다.
- 빈 상태, 오류 상태, 로딩 상태 문구를 시나리오별로 나눠서 사용자 이해를 돕는다.

### 2단계: 프로토타입 기능 확장

- 캠프 모집글 수정, 모집 마감, 상태 변경 흐름을 추가한다.
- 제출 초안과 최종 제출의 이력 관리 구조를 보강한다.
- 해커톤별 평가 기준 차이를 반영할 수 있도록 리더보드 모델을 세분화한다.

### 3단계: 서비스 구조 확장

- 인증과 계정 기반 저장을 도입해 브라우저 단위 상태를 서버 기반 상태로 확장한다.
- 백엔드 DB 와 쓰기 API 를 연결해 실제 서비스 운영이 가능하도록 한다.
- 관리자 화면, 운영 도구, 실시간 협업 기능은 범위 확장 이후 별도 트랙으로 추진한다.

## 운영 품질 개선 아이디어

- 배포 환경 기준의 수동 QA 시나리오를 문서화해 반복 검증한다.
- 문서와 구현 간 계약 변경 시점이 어긋나지 않도록 업데이트 체크리스트를 만든다.
- 제출 데이터와 결과물 링크 관리 기준을 정리해 공개 범위를 더 명확히 유지한다.
