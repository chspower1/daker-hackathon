# 페이지 구성

## 공통 구조

- 랜딩 페이지는 `app/page.tsx` 에서 시작한다.
- 앱 내부 페이지는 `app/(app)/layout.tsx` 와 `components/layout/SharedAppShell.tsx` 를 공유한다.
- 공통 셸은 상단 내비게이션, 언어 전환, `bootstrapStorage()` 실행을 담당한다.

## 페이지 연결 관계

- `/` -> `/hackathons`, `/camp`, `/rankings`
- `/hackathons` -> `/hackathons/[slug]`
- `/hackathons/[slug]` -> `/camp?hackathon={slug}`
- `/hackathons`, `/camp`, `/rankings` 는 공통 상단 내비게이션으로 서로 이동할 수 있다.

## 주요 페이지 설명

| 페이지 | 라우트 | 구현 위치 | 화면 설명 | 가능한 행동 |
|---|---|---|---|---|
| 랜딩 | `/` | `app/page.tsx` | 서비스 소개, 핵심 메시지, 주요 기능 요약을 보여주는 진입 화면이다. 해커톤과 캠프로 바로 이어지는 CTA를 배치한다. | `해커톤 탐색하기`, `캠프로 이동`, 랭킹 관련 CTA 클릭으로 다음 페이지로 이동한다. |
| 해커톤 목록 | `/hackathons` | `app/(app)/hackathons/page.tsx` | 공개 해커톤을 카드 또는 테이블 뷰로 보여준다. 상태 필터, 태그 필터, 키워드 검색을 통해 목록을 좁혀 볼 수 있다. 테이블 뷰에서는 주요 표시 컬럼(제목, 상태, 시작/종료일)을 기준으로 오름차순/내림차순 정렬이 가능하다. 키워드 검색은 목록 상단에 위치하며, 태그 영역은 사이드바 내부에서 스크롤되며 검색을 지원한다. | 상태 필터(건수 표시) 선택, 태그 다중 선택, 키워드 검색, 필터 초기화, 카드/테이블 뷰 토글, 컬럼 정렬, 상세 이동이 가능하다. |
| 해커톤 상세 | `/hackathons/[slug]` | `app/(app)/hackathons/[slug]/page.tsx` | 선택한 해커톤의 공개 정보를 한 화면에 모아 보여주는 중심 페이지다. 개요부터 팀, 제출, 리더보드까지 8개 섹션을 같은 화면에서 확인하며, 스크롤 위치에 따라 사이드 목차가 자동으로 강조(Scroll-spy)된다. 모바일에서는 현재 섹션 항목이 가로 스크롤 영역 안에서 자동으로 중심에 오도록 유지한다. | 섹션 앵커 이동(클릭 시 URL hash 갱신), `캠프로 이동`, 프로필 생성, 제출 초안 저장, 최종 제출이 가능하다. |
| 캠프 | `/camp` | `app/(app)/camp/page.tsx` | 팀 모집글 목록을 카드 또는 테이블 뷰로 제공하며 모집글 작성 모달을 지원한다. 특정 해커톤, 상태, 태그, 그리고 채용 정보(팀명, 소개, 역할 등) 대상 키워드 검색으로 목록을 좁힐 수 있다. 키워드 검색은 목록 상단에 위치하며, 태그 영역은 사이드바에 위치한다. 테이블 뷰의 모든 표시 컬럼은 정렬을 지원한다. | 카드/테이블 뷰 토글, 컬럼 정렬, 상태 필터 선택, 태그/키워드 검색, 프로필 및 모집글 작성, 연락 링크 열기가 가능하다. |
| 글로벌 랭킹 | `/rankings` | `app/(app)/rankings/page.tsx` | 시즌 기준 공개 랭킹을 표 형태로 보여주는 결과 확인 페이지다. 순위, 닉네임, 점수를 중심으로 구성한다. | 랭킹 확인, 새로고침 또는 저장소 갱신에 따른 최신 상태 확인이 가능하다. |

## 해커톤 상세 고정 섹션

해커톤 상세 페이지는 아래 8개 섹션을 완성 기준으로 본다.

1. 개요 (`overview`)
2. 안내 및 공지 (`info`)
3. 평가 (`eval`)
4. 일정 (`schedule`)
5. 상금 (`prize`)
6. 팀 (`teams`)
7. 제출 (`submit`)
8. 리더보드 (`leaderboard`)

## 페이지별 상태 처리 기준

- 해커톤 목록, 해커톤 상세, 캠프, 랭킹은 모두 로딩 상태와 데이터 없음 상태를 분리해서 보여준다.
- 목록/랭킹/상세 읽기에 실패하면 사용자에게 오류 상태 메시지를 노출한다.
- 상세 섹션 일부 정보가 비어 있더라도 페이지 전체를 막지 않고 해당 섹션만 빈 상태로 처리한다.
