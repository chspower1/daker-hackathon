# 검증 가이드

이 프로젝트는 자동 테스트 스크립트 대신 `pnpm lint`, `pnpm build`, 그리고 주요 사용자 흐름 기반의 수동 QA를 기본 검증 경로로 사용합니다.

## 공통 사전 검증

```bash
pnpm lint
pnpm build
```

- lint/build가 실패하면 수동 QA 전에 원인을 먼저 해결합니다.
- 수동 QA 중 저장소 동작을 확인해야 할 때는 브라우저 DevTools의 `localStorage`를 함께 확인합니다.

## 시나리오 1. 홈에서 해커톤 상세까지 이동

1. `/` 접속
2. `해커톤 탐색하기` 클릭
3. `/hackathons`에서 상태 필터, 태그 필터, 키워드 검색 적용
4. 카드 또는 테이블 행을 눌러 `/hackathons/[slug]` 진입

기대 결과

- 랜딩 CTA가 정상 이동한다.
- 필터 조합에 따라 결과 수와 빈 상태가 일관되게 바뀐다.
- 상세 페이지에서 개요/안내/평가/일정/상금/팀/제출/리더보드 섹션이 보인다.

## 시나리오 2. 상세에서 캠프로 이동하고 모집글 탐색

1. 해커톤 상세의 `캠프로 이동` 클릭
2. `/camp?hackathon={slug}`로 이동했는지 확인
3. 카드/테이블 뷰, 상태 필터, 태그 필터, 키워드 검색 사용

기대 결과

- 쿼리의 `hackathon` 값과 일치하는 모집글만 우선 노출된다.
- 필터 결과가 없으면 필터 전용 빈 상태가 보인다.
- 테이블 정렬과 뷰 전환이 깨지지 않는다.

## 시나리오 3. 로컬 프로필 생성 후 제출 저장

1. 해커톤 상세의 `Submit` 섹션 진입
2. 프로필이 없다면 닉네임으로 로컬 프로필 생성
3. 팀 이름, 산출물 링크, 메모 입력 후 `Save Draft`
4. 다시 열어 `Final Submit`

기대 결과

- 프로필 생성 후 같은 브라우저에서 제출 가능 상태가 된다.
- draft 저장 후 상태와 마지막 수정 시간이 보인다.
- final submit 후 리더보드에 반영된다.
- `localStorage.submissions`, `localStorage.leaderboards` 값이 논리적으로 일치한다.

## 시나리오 4. 랭킹과 언어 전환 유지

1. `/rankings` 접속
2. 언어 전환 UI에서 한국어/영어를 번갈아 선택
3. 새로고침 수행
4. 페이지를 열어 둔 상태에서 DevTools 또는 다른 탭으로 `localStorage.rankings` 값을 바꾼다.

기대 결과

- 페이지 제목, 설명, 표 헤더, 주요 문구가 선택 언어에 맞게 바뀐다.
- 새로고침 후에도 마지막 언어 선택이 유지된다.
- 페이지를 다시 열지 않아도 최신 랭킹 값이 반영된다.
- 랭킹 데이터가 없을 때는 빈 상태, 읽기 실패 시 오류 상태가 표시된다.

## 시나리오 5. 다크모드 전환과 새로고침 유지

1. `/` 또는 `/hackathons` 에 접속
2. 상단 헤더의 테마 토글을 눌러 다크모드로 전환
3. 랜딩 또는 앱 화면의 헤더, 카드, 입력창, 모달 배경/텍스트 대비 확인
4. 새로고침 수행 후 같은 테마가 유지되는지 확인

기대 결과

- 상단 헤더 토글로 라이트/다크 테마가 즉시 전환된다.
- 주요 표면 색, 텍스트 색, 경계선이 다크 팔레트에 맞게 읽기 쉬운 대비를 유지한다.
- 새로고침 후에도 마지막 테마 선택이 유지된다.
- `localStorage.app-theme` 값과 현재 렌더링된 테마가 일치한다.

## 시나리오 6. 헤더 모의 로그인과 로그아웃

1. `/hackathons` 또는 `/hackathons/[slug]` 에 접속
2. 헤더의 `로그인` 버튼 클릭
3. prompt 에 프로필 ID를 입력
4. 헤더에 입력한 프로필 ID와 `로그아웃` 버튼이 보이는지 확인
5. 새로고침 후 같은 프로필 ID가 유지되는지 확인
6. `로그아웃` 버튼 클릭

기대 결과

- 입력한 프로필 ID가 `localStorage.localProfile` 의 `id`, `nickname` 으로 저장된다.
- 새로고침 후에도 헤더의 로그인 상태가 유지된다.
- `로그아웃` 후 헤더는 다시 `로그인` 버튼 상태로 돌아간다.
- 프로필이 필요한 화면을 열어 둔 상태라면 새로고침 없이도 프로필 요구 상태로 즉시 전환된다.
