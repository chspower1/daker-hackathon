# Daker Hackathon 제출 준비

## 1. 문서 목적

이 문서는 제출 전 점검 항목을 정리한다. 상세 기능 사양은 각 문서로 분리했고, 여기서는 심사자가 바로 확인할 수 있는 체크리스트만 남긴다.

## 2. 필수 제출물

- 기획서
- 웹 페이지
- Vercel 배포 URL
- GitHub 저장소
- 솔루션 PDF

## 3. 문서 연결

- 제품 개요: [`docs/index.md`](index.md)
- 범위/규칙: [`docs/common/product-and-scope.md`](common/product-and-scope.md)
- 기능 상세: [`docs/features/*`](features/)
- 수용 기준: [`docs/common/acceptance.md`](common/acceptance.md)

## 5. 심사자 데모 시나리오(요지)

1. 메인에서 핵심 CTA 3개 확인
2. 해커톤 목록 → 상세 → 제출 저장/제출
3. `/camp`에서 모집글 작성, 새로고침 유지 확인
4. `/rankings`에서 공개 닉네임 기준 랭킹 확인

## 6. 제출 전 체크리스트

- [ ] 필수 제출물 5종이 모두 준비되었다.
- [ ] Vercel URL과 GitHub 저장소 링크가 외부에서 정상 접근된다.
- [ ] 심사자 데모 시나리오를 실제 배포 환경에서 끝까지 확인했다.
- [ ] 제품 내부 품질 검수는 `docs/common/acceptance.md` 기준으로 완료했다.
- [ ] 기획서, 기능 문서, 실제 화면의 용어가 일치한다.
- [ ] 비공개/민감 정보가 노출되지 않는다.

## 7. 금지 항목

- 비공개/민감 정보 노출
- 범위 바깥 기능을 필수로 제시
- 기획서와 실제 구현 불일치
