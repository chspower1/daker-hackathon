# 공통 수용 기준

## 1. 완성도 기준

기능 완성 판단은 아래를 모두 충족해야 한다.

- 필수 라우트 동작
- 공개 데이터 렌더링
- 로딩/빈/에러 상태 처리
- 로컬 프로필 생성/재사용
- 팀 모집/제출 데이터의 새로고침 지속성
- 리더보드 반영 일관성
- 공개 정보만 표시

## 2. 기능군별 수용 기준

### 2.1 내비게이션

- 상단 네비게이션에서 핵심 화면 이동 가능
- 홈, 목록, 상세, 캠프, 랭킹 간 이동 흐름 끊김 없음

### 2.2 데이터 정합성

- 공개 데이터와 로컬 데이터만 렌더링
- 숨겨진 내부 필드 노출 없음

### 2.3 로컬 프로필

- 읽기 화면은 프로필 없이도 동작
- 쓰기 전 프로필 생성 또는 재사용 조건 충족
- 새로고침 후에도 동일 프로필 기반 데이터 유지

### 2.4 영속성

- 팀 모집글 생성 후 새로고침 유지
- 제출 저장/제출 후 상태 유지
- 리더보드 반영 유지

### 2.5 화면 플로우

- 홈 → 목록 → 상세 이동 가능
- 상세 핵심 섹션 전부 렌더링
- 팀 모집 작성 및 제출 전 프로필 확인 동작

### 2.6 랭킹

- `rank`, `nickname`, `points` 컬럼 표시
- 글로벌 닉네임 기준 노출
- 내부 계정 노출 없음

### 2.7 다국어

- 한국어/영어 전환 UI가 동작한다.
- 언어 전환 후 주요 카피, CTA, 표 헤더, 접근성 문구가 선택 언어로 표시된다.
- 새로고침 후에도 마지막 선택 언어가 유지된다.
- `<html lang>`와 메타데이터가 현재 언어 기준과 일치한다.

## 3. 공통 예외 처리

- 파싱 실패: 기본 시드 또는 에러 상태로 안전 처리
- 외부 데이터 누락: 빈 상태 메시지와 행동 유도 노출
- 사용자 입력 오류: 명확한 안내 후 재입력 유도

## 4. 테스트 포인트

각 기능 문서는 아래 항목을 별도로 검증한다.

- 홈: 핵심 CTA 가시성, 이동 성공
- 해커톤 목록: 모든 항목 노출, 상세 이동, 필터 동작
- 상세: 8개 섹션 렌더링, 제출 폼 가변 처리
- 캠프: 전체/해커톤별 필터, 생성 플로우, 데이터 유지
- 랭킹: 열거형 컬럼, 공개 닉네임 정합성
