# 서비스 개요

## 서비스명

- `HackPlatform`

## 한줄 소개

- 공개 해커톤을 탐색하고, 팀원을 모집하고, 결과 제출과 랭킹 확인까지 한 흐름으로 이어서 사용할 수 있는 해커톤 플랫폼 프로토타입이다.

## 개발 목표

- 해커톤 탐색 -> 팀 빌딩 -> 제출 -> 결과 확인으로 이어지는 핵심 사용자 흐름을 프론트엔드만으로 끝까지 검증한다.
- 백엔드 없이도 `docs/requirements/예시자료/` 와 `localStorage` 를 이용해 실제 사용에 가까운 상태 유지 경험을 제공한다.
- 한국어/영어 전환, 새로고침 유지, 빈 상태/오류 상태까지 포함해 서비스 이해가 가능한 기준 화면을 완성한다.

## 주요 사용자와 제공 가치

| 사용자 | 사용하는 이유 | 제공 가치 |
|---|---|---|
| 해커톤 참가자 | 참여할 이벤트를 찾고 비교하려고 함 | 목록과 상세 화면에서 일정, 규칙, 평가 기준, 상금, 팀 현황을 한 번에 확인할 수 있다. |
| 팀원을 찾는 개인/소규모 팀 | 함께 참가할 팀원을 구하려고 함 | `Camp` 에서 모집글을 탐색하고, 자신의 모집글을 직접 작성해 연결을 시작할 수 있다. |
| 제출을 준비하는 참가자 | 결과물을 저장하고 최종 제출하려고 함 | 로컬 프로필을 만든 뒤 제출 초안을 저장하고, 최종 제출 시 리더보드 반영까지 이어서 확인할 수 있다. |
| 결과를 확인하는 사용자 | 시즌 성과와 순위를 확인하려고 함 | 해커톤별 리더보드와 글로벌 랭킹 화면에서 공개 가능한 결과를 빠르게 볼 수 있다. |

## 구현 범위

- 랜딩 페이지에서 핵심 기능 소개와 주요 페이지 진입 제공
- `/hackathons` 에서 공개 해커톤 목록 탐색 및 필터링 제공
- `/hackathons/[slug]` 에서 해커톤 상세 정보, 팀 현황, 제출, 리더보드 제공
- `/camp` 에서 팀 모집글 탐색 및 생성 제공
- `/rankings` 에서 글로벌 랭킹 조회 제공
- 한국어(`ko`) / 영어(`en`) 전환과 새로고침 후 상태 유지 제공

## 서비스 운영 전제

- 읽기 기능은 로컬 프로필 없이도 사용할 수 있다.
- 쓰기 기능은 `localProfile` 이 준비된 뒤에만 허용한다.
- 공개 읽기 데이터는 `docs/requirements/예시자료/` 의 JSON을 기준으로 하고, 사용자 생성 데이터는 브라우저 `localStorage` 에 저장한다.
- 민감 정보, 내부 식별자, 운영용 데이터는 화면에 노출하지 않는다.

## 주요 라우트

- `/`: 서비스 소개와 핵심 CTA를 제공하는 랜딩 페이지
- `/hackathons`: 해커톤 목록 탐색 페이지
- `/hackathons/[slug]`: 선택한 해커톤의 상세 페이지
- `/camp`: 팀 모집글 탐색 및 작성 페이지
- `/rankings`: 글로벌 랭킹 페이지

## 제외 범위

- 회원가입, 로그인, 외부 인증 연동
- 백엔드 DB, 쓰기 API, 서버 동기화
- 관리자 화면, 비공개 심사 화면, 운영 도구
- 실시간 채팅, 알림, 메시징
- 계정 기반 다기기 동기화
